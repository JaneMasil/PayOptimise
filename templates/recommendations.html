Hybrid scoring formula
# recommender.py — 4-signal hybrid scoring WEIGHTS = { 'collaborative': 0.40, # historical preference from feature store 'device_match': 0.31, # mobile=1-tap methods, desktop=bank/card 'time_context': 0.22, # peak hours: 12-14, 19-21 → 1.0x; else 0.75x 'session_depth': 0.07, # page_depth / 5.0 → intent proxy } score = (WEIGHTS['collaborative'] * collab_signal # past preference + WEIGHTS['device_match'] * device_pref # mobile/desktop affinity + WEIGHTS['time_context'] * (1.0 if is_peak else 0.75) + WEIGHTS['session_depth'] * min(depth/5, 1.0))
Why hybrid?
Pure collaborative filtering scores 17.9% CTR. Adding device + time context lifts this to 24.3–27.4%. The 8.4% additional gain comes from real-time feature store signals. (Yin et al., 2025)
Shared feature store
Real-time session context (current funnel stage, time-on-page) flows from the analytics dashboard into the recommendation engine. This cross-component integration is what answers RQ2.
FinTech specifics
Unlike entertainment recommendations, financial recommendations require interpretability. Each score includes reason strings. Future work: SHAP values for regulatory compliance. (Mendonça et al., 2023)
